# CHECKLIST-B0 — Δοκιμαστική λειτουργία (2 εβδομάδες)

Για: **Χρήστο, Λωίδα, Γιολάντα**. Στόχος: να τρέξει το Constructor την πραγματική
δουλειά της ομάδας για 2+ εβδομάδες και να βγει **μία πραγματική αναφορά σε έναν
πραγματικό κατασκευαστή**. Ό,τι δεν δουλεύει καλά, γράφεται στο «Ημερολόγιο τριβών»
στο τέλος αυτού του αρχείου — αυτή η λίστα θα καθορίσει τι θα φτιαχτεί μετά.

## Άπαξ — πρώτη μέρα (ο Χρήστος, στον υπολογιστή του γραφείου)

```bash
cd "/Users/chrism/Project with Claude/CONSTRUCTOR"

# 1. Διάλεξε το PIN της ομάδας (4–12 ψηφία) και πρόσθεσέ το στο .env:
#      CONSTRUCTOR_PIN=XXXX
#    (μία γραμμή στο τέλος του αρχείου .env — ο Χρήστος το επιλέγει, δεν το μοιραζόμαστε γραπτά αλλού)

# 2. Βρες τη διεύθυνση του Mac στο Wi-Fi του γραφείου:
ipconfig getifaddr en0        # π.χ. 192.168.1.23

# 3. Στήσιμο δεδομένων + εκκίνηση:
bun run db:init
bun run seed <αρχείο-με-το-τρέχον-pipeline>.json   # δείτε το seed.example.json για τη μορφή
CONSTRUCTOR_HOST=0.0.0.0 bun run dev                # αφήστε το να τρέχει
# (χωρίς PIN στο .env ΔΕΝ ξεκινά εκτεθειμένο — σκόπιμα· βάλε πρώτα το PIN)
```

- [ ] Φορτώθηκε το τρέχον pipeline (ο πίνακας ΔΕΝ είναι άδειος).
- [ ] Το αυτόματο backup είναι ενεργό (τρέχει κάθε βράδυ 21:00 — έλεγχος:
      `tail -3 ~/constructor-backups/backup.log`).
- [ ] Σελιδοδείκτης σε ΚΑΘΕ κινητό: `http://<IP-του-Mac>:3000` (π.χ. http://192.168.1.23:3000)
      — δουλεύει μόνο στο Wi-Fi του γραφείου, με το PIN της ομάδας.
- [ ] Δοκιμή από ένα κινητό: άνοιγμα → PIN → «Ποιος είσαι;» → πίνακας.

## Καθημερινά — κάθε χειριστής

1. **Άνοιγμα εφαρμογής (στο Wi-Fi του γραφείου) → PIN ομάδας → «Ποιος είσαι;» →
   το όνομά σου.** (Μία φορά ανά συνεδρία· αν αλλάξεις συσκευή, ξανά.)
   Εκτός γραφείου δεν έχει πρόσβαση κανείς — ούτε εμείς· ό,τι γίνεται στο πεδίο
   καταχωρείται με την επιστροφή (και αν αυτό ενοχλεί, γράψτε το στο Ημερολόγιο τριβών!).
2. **Κάθε επαφή καταχωρείται ΤΗΝ ΙΔΙΑ ΜΕΡΑ** — τίποτα σε χαρτάκια/μνήμη:
   - Νέο ενδιαφέρον → **Νέος** (πηγή, προφίλ, budget, επόμενη ενέργεια).
   - Έγινε επίσκεψη → **Επίσκεψη** (μονάδα, αγοραστής, βαθμός 1–5, επόμενη ενέργεια).
   - Ήρθε προσφορά → **Προσφορά** (μονάδα, αγοραστής, ποσό — δείτε την πρόταση
     αντιπρότασης, με ένα άγγιγμα γίνεται επόμενη ενέργεια).
3. **Η «επόμενη ενέργεια» είναι υποχρεωτική και ΑΛΗΘΙΝΗ** — γράψε τι πραγματικά
   θα κάνεις (όχι «follow up»). Το κουμπί δεν ανάβει χωρίς αυτήν — σκόπιμα.
4. **Πρωινό βλέμμα στον πίνακα**: οι πρώτες κάρτες είναι ό,τι χρειάζεται προσοχή.
   Δούλεψε από πάνω προς τα κάτω.
5. **Στοιχεία επικοινωνίας αγοραστών** (τηλέφωνα, email) μένουν εκεί που είναι
   σήμερα (ilist/ατζέντα) — η εφαρμογή δουλεύει με ψευδώνυμα (#1, #2…). Μην τα
   γράφετε σε σημειώσεις ή στην «επόμενη ενέργεια».
6. **Κάτι σε δυσκόλεψε/καθυστέρησε; Γράψτο στο Ημερολόγιο τριβών** — με μία
   γραμμή, την ίδια μέρα. Και τα «ψιλά» μετράνε (ένα κουμπί που ψάχνεις, ένα
   πεδίο που λείπει, κάτι που θες να βλέπεις στον πίνακα).

## Κάθε δεύτερη Παρασκευή — η αναφορά (Χρήστος)

```bash
bun run report --builder="<όνομα>" --project="<έργο>" --period=biweekly --html > αναφορά.html   # για τον κατασκευαστή
bun run report --builder="<όνομα>" --project="<έργο>" --period=biweekly --brief                  # τα σήματα για τα insights
```

- [ ] Άνοιξε το Claude Code στον φάκελο του project → τρέξε `/insights` → επικόλλησε το brief.
- [ ] Διάβασε τις 2–3 προτάσεις, διόρθωσε ό,τι δεν στέκει, επικόλλησέ τις στην αναφορά.
- [ ] **Άνοιξε το αναφορά.html και διάβασέ το όπως ο κατασκευαστής.** Στέκει; Επισύναψέ το σε email/Viber και στείλ' το.
- [ ] Σημείωσε στο Ημερολόγιο: πόση ώρα πήρε όλο το βήμα, και την αντίδραση του κατασκευαστή.

## Τι μετράμε (κριτήρια εξόδου — χωρίς αυτά δεν προχωράμε σε νέα features)

- ≥ 2 εβδομάδες καθημερινής χρήσης και από τους τρεις.
- ≥ 1 αναφορά που στάλθηκε πραγματικά σε κατασκευαστή.
- Καταχωρήσεις ανά ημέρα ανά χειριστή (φαίνεται με: `bun run report --separation`).
- Συμπληρωμένο Ημερολόγιο τριβών.

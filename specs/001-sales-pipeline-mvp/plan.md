# Implementation Plan — 001 · Sales Pipeline MVP

## Technical context

- **Runtime**: Bun (latest). **Language**: TypeScript.
- **Storage**: `bun:sqlite` (built-in), single file `constructor.db`. No ORM (Article VIII).
- **Server**: `Bun.serve` — a thin HTTP/JSON API. No framework unless justified.
- **Frontend**: React (reuse the existing capture prototype), bundled with Bun's bundler. Minimal deps: react, react-dom, lucide-react. No CSS framework required (inline styles as in prototype); Tailwind optional.
- **Report output**: Markdown → optional HTML. PDF is a nice-to-have, not required for prototype.
- **AI edge (Article III, Max-native)**: NO in-app API call. `report` emits an insight brief; insight prose is generated by an interactive Claude Code slash-command (`.claude/commands/insights.md`) run by the operator under the Max subscription, then pasted into the report. Rationale: headless/in-app `claude -p` is not reliably Max-covered, and wrapping Claude Code in a product is steered to API keys — so the AI step stays interactive and human-run, with no API key in the system.
- **Testing**: `bun test` (Bun's built-in test runner). Test-first (Article IX).
- **Deploy**: local-first for the prototype (Bun server + local `constructor.db`). Hosting is out of scope; note the seam for later.

## Phases

**Phase 0 — Foundation.** Repo + package.json + bun scripts; apply schema; domain pure functions (temperature/counter/recommend) test-first. No UI yet.

**Phase 1 — Capture + pipeline (US-1..US-4).** DB queries; Bun.serve capture endpoints with next_action enforcement; wire the React board + capture sheets to the API; seed/migrate command (US-7).

**Phase 2 — Reporting (US-5, US-6).** Deterministic biweekly builder (Greek); monthly/quarterly extensions; the "no naked bad number" recommendation pairing (unit-tested); the insight-brief emitter (deterministic) + the interactive Claude Code `/insights` command; `bun run report` CLI.

## Report command contract

```
bun run report --builder="<name>" --project="<project_name>" --period=biweekly|monthly|quarterly \
  [--anchor=DATE] [--rolling] [--as-of=DATE] [--brief]
# Default: FIXED non-overlapping window anchored to --anchor (default = project.listed_at).
#          Consecutive reports never double-count.
# --rolling : last N days from today (internal "how are we doing now"; not for the client cadence).
# --as-of=DATE : compute as if run on DATE (reproducibility, Article III).
# --brief : also emit the structured insight brief → feed it to the Claude Code /insights command (Max).
# stdout: Greek Markdown report. Exit 0 on success. NO LLM API is ever called in-app.
# Deterministic: identical DB state + same flags ⇒ identical numbers.
```
